## Project Description
In this exercise, I set up a development environment using ESLint and Prettier to improve code quality and consistency in my To-Do List project.
ESLint was used to detect errors, enforce coding standards, and highlight bad practices, while Prettier automatically formatted the code to maintain a clean and consistent style. I then ran linting on my project and fixed all identified issues

## Technologies Used
- HTML5
- CSS3
- JavaScript
  

## Features
- Code quality checking.
- Automatic code formatting
- Error prevention rules

## How to Run
1. Clone this repository
2. Open `index.html` in your browser
   OR
   Run `npm install` then `npm start`

## Lessons Learned
- ESLint helps catch bugs early, especially logical errors that are easy to miss.
- Prettier ensures consistent code formatting, improving readability.
- Using === instead of == prevents unexpected type-related bugs.
- Writing clean code improves teamwork and maintainability in real projects.
- Automation tools like linting save time and reduce manual debugging.

## Challenges Faced
- Understanding ESLint rules like eqeqeq, curly, and prefer-const at first was confusing.
- Dealing with multiple warnings vs errors and knowing which ones to fix immediately.
- Configuring ESLint and Prettier to work together without conflicting formatting rules.
- Remembering to run lint commands from the correct project directory.
- Fixing small but important issues like missing strict equality (===) and unused variables.
